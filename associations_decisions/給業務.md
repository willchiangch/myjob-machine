## 向五公會提案說詞

> **背景：** 4/22、4/29 教學後觀察到會務人員與部分長官對新介面與新流程的抗拒程度高；習慣舊系統的會員反彈預期更強。為避免移轉計畫讓公會對外受到指責，以「在凌群服務雲建至會計師專區，將E化系統整併架構上線」作為主訴求。

### 對客戶的核心訴求

> 「一樣是搬遷至凌群服務雲，但我們改為把現有系統調整架構，而功能與介面流程原封不動整併搬進來，公會節費目標達成，會員完全不需要換介面。」

### 對公會的好處（依說服優先序）

| # | 說法重點 | 備註 |
|:--|:--------|:-----|
| 1 | **合約金額不變，確實達到節費目標** | 年租金不動，GCP 費用下降，公會帳面受益清楚 |
| 2 | **會員不需適應新介面、新流程** | 搬遷後操作方式完全相同，僅需驗收功能正常，不會引發會員投訴 |
| 3 | **網域不換** | 各公會的網址不變，不再需要考慮轉址問題以及過往印鑑證明QR Code失效問題 |
| 4 | **App 不需重新下載** | Android / iOS / React Native 三個 App 無需重新發版 |
| 5 | **未來更新不再需要停機維護** | Cloud Run 架構支援零停機部署（Rolling Update） |
| 6 | **直接簽維運服務合約，不再為點數制煩惱** | 每年固定的吃到飽服務費，不用擔心點數問題 |
| 7 | **台中公會理事長在意的五公會單一帳號登入成為可能** ⭐ | 整合後 Redis + JWT 架構為跨組織 SSO 鋪路；**需先向主管機關確認可行性，確認前作為籌碼保留，不主動承諾** |

### 已知缺點（坦誠說明，建立信任）

| 缺點 | 說明 |
|:----|:-----|
| 無新功能（如新公文會辦機制、長官帳號整併等） | 後續功能需另簽 CR |
| SSL 憑證與 SendGrid 費用仍由公會自行負擔 | 日後可改為 GCP 代管憑證（免費、自動續約）；作為後續升級誘因保留，切換前不主動承諾 |
| 現有問題的優化（如報表效能）需另行 CR | 搬遷本身已修復取號競態等 CRITICAL bug，其餘優化清單可附上作為後續合作基礎 |
